# 🖼️ 图床

基于 GitHub + jsDelivr 的个人图床工具。

## 功能特性

- ✅ GitHub 仓库存储
- ✅ jsDelivr CDN 加速
- ✅ 拖拽上传
- ✅ 剪贴板粘贴
- ✅ 上传历史管理
- ✅ 从仓库同步

## 使用方法

### 配置 GitHub

**前置要求**：
1. GitHub 账号
2. 创建一个公开仓库用于存储图片
3. 生成 Personal Access Token（需要 repo 权限）

**配置步骤**：
1. 打开图床工具
2. 输入 GitHub 用户名
3. 输入仓库名称（如：`images`）
4. 输入分支名（如：`main`）
5. 输入 Personal Access Token
6. 设置存储路径（如：`images`）
7. 点击验证配置
8. 保存配置

### 上传图片

**方式 1：拖拽上传**
1. 打开图床工具
2. 拖拽图片到上传区域
3. 等待上传完成
4. 自动复制链接

**方式 2：粘贴上传**
1. 复制图片到剪贴板
2. 打开图床工具
3. 按 `Cmd/Ctrl+V` 粘贴
4. 自动上传

**方式 3：选择文件**
1. 点击「选择图片」
2. 选择图片文件
3. 上传

### 获取链接

上传成功后可以获取：
- **直接链接**：原始图片 URL
- **Markdown**：`![](url)` 格式
- **HTML**：`<img src="url">` 格式
- **BBCode**：`[img]url[/img]` 格式

## jsDelivr CDN

### CDN 加速

上传的图片通过 jsDelivr CDN 访问：
```
https://cdn.jsdelivr.net/gh/用户名/仓库@分支/路径/图片.jpg
```

**优势**：
- 全球 CDN 加速
- 无流量限制
- 永久有效
- 访问速度快

### 版本控制

使用分支作为版本：
- `@main`：最新版本
- `@v1.0.0`：特定标签

## 隐私说明

### 公开访问

- GitHub 仓库必须公开
- 上传的图片公开可访问

### Token 安全

- Token 本地加密存储
- 不会上传到服务器
- 定期更换 Token

## 常见问题

### Q: 上传失败？
**A**:
1. 检查 Token 是否有效
2. 确认仓库存在且为公开
3. 检查分支名称是否正确
4. 验证网络连接

### Q: 图片加载慢？
**A**:
1. jsDelivr 在国内可能较慢
2. 等待 CDN 缓存生效
3. 使用代理加速

### Q: 仓库满了怎么办？
**A**:
1. GitHub 单仓库 1GB 软限制
2. 删除不需要的图片
3. 创建新仓库继续使用

### Q: 如何迁移图片？
**A**:
1. 使用「从仓库同步」功能
2. 或手动下载后重新上传

## 技术细节

### 上传流程

```
选择图片
    ↓
Base64 编码
    ↓
调用 GitHub API
    ↓
上传到仓库
    ↓
生成 jsDelivr CDN 链接
    ↓
保存到历史记录
```

### API 调用

使用 GitHub Contents API：
```
PUT /repos/{owner}/{repo}/contents/{path}
```

### 数据存储

本地存储位置：`~/.ltools/imagebed/`
- 配置文件：`config.json`
- 历史记录：`history.json`
